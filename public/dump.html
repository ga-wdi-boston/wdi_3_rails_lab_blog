Routes

The first thing that happens when you make a request on a rails app is Routing. The router takes the HTTP request (url & method) directs to proper Controller and Method.

Controller

The controller's purpose is to deal with user input, and the user input comes in as params. Then the controller interacts with the model. Then the controller interacts with view, returning the view instance variables of the model.

Model

The models are object representations of the data base schema. It provides behavior and rules for the data. The model is all about data. Doesn't know anything about html, css, erb files, views, input from the controller, it just knows about its data. The model ties into the database.

View

The view mixes data together from the controller with html, rendering out the information sent from the controller. Also for creating forms that can make more requests.

Migrations

Migrations change the structure of the database. They are not triggered by HTTP requests (they are run manually).

The HTTP requests go like this with Rails app:

  1. Browser makes GET request to the server
  2. Routes look at URL + HTTP method direct to matching controller and action
  3. Controller action
    • Take optional user input from params
    • Use the model to query the database with user input
    • Store data in @instance variable
  4. Name matching ERB view: compile/render info from @instance variables into HTML
  5. End of controller action returns the rendered HTML as a response.
  6. Browser displays HTML

Rails migrations

rails g migration CreateBooks title author_name price:decimal creates a new table on your database called Books

rake db:migrate runs the migration (adds the new table `Books`

rails g migration AddNumberOfPagesAndLanguageToBooks num_pages:integer language:string adds two new columns called num_pages which takes an integer, and language, which takes a string.

rake db:migrate to apply the migration (actually adds the columns to the tables)

rake db:rollback rolls back your migration by one migration


class CreateBooks
def change
    create_table :books do |t|
      t.string :title
      t.string :author #=> added in after creating the initial migration table
      t.date :pub_date #=> added in after creating the initial migration table
      t.timestamps null: false
    end
  end
end


by rolling back before your initial migration, you can alter the original schema in your ruby file.


rake db:migrate to apply your changes to your db
